# Cover Letter Skill

신입 공채 자기소개서를 5명의 전문가 피드백 루프를 통해 작성하는 스킬.

## 인수 (Arguments)

| 형식 | 설명 | 예시 |
|------|------|------|
| URL | 공고 페이지에서 질문 항목 자동 추출 | `/cover-letter https://recruit.example.com/job/123` |
| 회사명 직군명 | 수동 입력 모드 | `/cover-letter 삼성전자 백엔드개발` |
| 없음 | 대화형 모드로 진입 | `/cover-letter` |

---

## 경로 정보

| 항목 | 경로 |
|------|------|
| vault | `{OBSIDIAN_VAULT}` (CLAUDE.md 또는 환경변수에서 설정) |
| 프로필 문서 | `{vault}/career/profile.md` |
| 기존 자소서 | `{vault}/career/` 하위 폴더 |
| 출력 경로 | `{vault}/career/{회사명 or 업종}/` |
| 출력 파일명 | `{공고마감일}-{회사명}-{직군명}.md` |
| AI 탐지 회피 가이드 | `references/anti-ai-detection.md` |

---

## 실행 절차

### Phase 1: 초기화

#### 1-1. 인수 파싱
- URL이면: WebFetch로 공고 페이지를 가져와 자소서 질문 항목, 글자 수 제한, 마감일, 회사명, 직군명 추출
- 회사명+직군명이면: 사용자에게 질문 항목과 글자 수 제한, 마감일을 대화로 수집
- 없으면: 대화형으로 하나씩 수집

#### 1-2. 프로필 문서 로드 (폴백 체인)
1. `{vault}/career/profile.md` 존재 확인 --> 있으면 Read로 로드
2. 프로필 없으면 --> 사용자에게 알리고 대화로 프로필 생성 제안:
   ```
   프로필 문서가 아직 없습니다. 지금 기본 프로필을 만들까요?
   (학력, 전공, 프로젝트, 기술스택, 경험, 강점/약점 등을 대화로 수집하여 profile.md 생성)
   ```
3. 프로필 생성을 거부하면 --> 기존 자소서 검색 (`{vault}/career/` 하위 .md 파일들)

#### 1-3. 문체 학습
기존 자소서나 개인 글이 있으면 분석하여 문체 특징 추출:
- 종결어미 패턴 (습니다/했다/했습니다 비율)
- 평균 문장 길이
- 선호 접속사/연결어
- 감정 표현 방식

문체 분석 결과를 내부 컨텍스트에 저장하여 작성 시 반영.

---

### Phase 2: 항목별 초안 작성 (병렬)

자소서 질문 항목이 N개이면, **최대 3개씩 병렬**로 서브 에이전트에게 초안 작성 위임.

각 서브 에이전트에 전달할 컨텍스트:
- 해당 질문 항목 + 글자 수 제한
- 프로필 문서 내용 (해당 질문에 관련된 부분 발췌)
- 회사/직군 정보
- 문체 분석 결과
- AI 탐지 회피 가이드 (`references/anti-ai-detection.md`)

서브 에이전트 핵심 지시:
1. `references/anti-ai-detection.md`를 반드시 Read하여 AI 탐지 회피 규칙 숙지
2. 사용자 문체 패턴에 맞춰 작성
3. 프로필 문서의 팩트만 사용 (날조 금지)
4. 구체적 경험이 부족하면 `[TODO: 본인 경험으로 대체 - 예: XX 프로젝트에서 겪은 구체적 에피소드]` 마커 삽입
5. 글자 수 제한 준수 (+-10% 허용, 기본 500~800자)
6. 자소서 본문은 plain text (마크다운 서식 없음)

---

### Phase 3: 5명 전문가 피드백 루프

초안 완성 후, 5명의 전문가가 동시에 평가. 모든 항목이 100점이 될 때까지 반복.

#### 전문가 구성

| 전문가 | 평가 항목 | 기준 |
|--------|----------|------|
| **AI 탐지 전문가** | AI 작성 탐지 회피 | `references/anti-ai-detection.md` 기준. AI 특유 패턴 잔존 여부 |
| **팩트체커** | 팩트 정확성 | 프로필 문서와 대조. 없는 경험 날조 여부, 수치 정확성 |
| **인사담당자** | 적절성/설득력 | 해당 회사/직군에 맞는 내용인가, 질문 의도에 부합하는가 |
| **문체 전문가** | 자연스러움 | 사용자 문체와 일치하는가, 자연스러운 한국어인가 |
| **구조 전문가** | 구성/흐름/글자수 | 논리적 흐름, 두괄식/미괄식 적절성, 글자 수 준수 |

#### 피드백 루프 절차

```
LOOP (max 5회):
  1. 5명 전문가 동시 평가 (병렬 에이전트, sonnet 모델)
     - 각 전문가가 항목별로 100점 만점 채점 + 구체적 수정 지시 반환
  2. 모든 항목 모든 전문가 100점이면 --> BREAK
  3. 100점 미만 항목이 있으면:
     a. 수정 지시를 반영하여 해당 부분만 재작성
     b. 사용자 입력이 필요한 경우 (팩트 부족, 경험 부재 등):
        --> 사용자에게 질문 + 선택지 제시
        --> 사용자가 스킵(Enter)하면 [TODO] 마커 유지
     c. 다음 라운드로
  4. 5회 초과 시 현재 상태로 확정 + 미달 항목 WARNING 표시
```

사용자 입력 요청 형식:
```
[Round 2/5] AI 탐지 전문가 피드백: 92/100
- 3번째 단락 "다양한 경험을 통해 성장했습니다" --> AI 특유 패턴 잔존

수정 제안: 구체적 경험 한 가지로 대체 필요.
--> 대학/프로젝트에서 겪은 구체적 에피소드가 있다면 알려주세요.
--> 없으면 Enter를 누르면 [TODO] 마커로 대체합니다.
```

---

### Phase 4: 최종 문서 생성

#### 4-1. 출력 경로 결정
회사명 기반으로 `{vault}/career/` 하위에 적절한 폴더 생성:
- IT 대기업: `career/대기업/`
- 스타트업: `career/스타트업/`
- 금융/보험: `career/금융/`
- 기타: `career/기타/`

파일명: `{공고마감일}-{회사명}-{직군명}.md`
- 예: `260401-삼성전자-백엔드개발.md`
- 마감일 모르면: `000000-삼성전자-백엔드개발.md`

#### 4-2. 문서 구조

```markdown
---
company: {회사명}
position: {직군명}
deadline: {마감일}
status: draft / final
created: {작성일}
tags:
  - career/cover-letter
  - career/{회사명}
---

# {회사명} - {직군명} 자기소개서

## 공고 정보
- 회사: {회사명}
- 직군: {직군명}
- 마감: {마감일}
- 출처: {공고 URL 또는 "수동 입력"}

---

## 항목 1: {질문}
> 글자 수 제한: {N}자 | 현재: {M}자

{자소서 내용 - plain text}

---

## 항목 2: {질문}
> 글자 수 제한: {N}자 | 현재: {M}자

{자소서 내용}

---

(항목 수만큼 반복)

---

## 전문가 평가 요약

| 전문가 | 항목1 | 항목2 | ... |
|--------|-------|-------|-----|
| AI 탐지 | 100 | 100 | |
| 팩트 | 100 | 100 | |
| 적절성 | 100 | 100 | |
| 문체 | 100 | 100 | |
| 구조 | 100 | 100 | |

## TODO 항목 (본인 수정 필요)
- [ ] 항목 2, 3번째 단락: [TODO: 설명]
```

#### 4-3. 완료 메시지
```
자기소개서 작성 완료!

저장 위치: career/{폴더}/{파일명}
- 항목 수: N개
- 전문가 평가: AI탐지 100 / 팩트 100 / 적절성 100 / 문체 100 / 구조 100
- TODO 항목: X건 (본인 수정 필요)

TIP: TODO 항목을 본인 경험으로 채우면 AI 탐지 회피가 더 강화됩니다.
```

```markdown
- 병역:

## 기술 스택
- 언어:
- 프레임워크:
- 도구:
- 자격증:

### 프로젝트 1: {이름}
- 기간:
- 역할:
- 기술:
- 성과: (수치로)
- 어려웠던 점 + 해결 방법:

## 대외활동/인턴

## 강점/약점
- 강점:
- 약점 (개선 노력 포함):

## 지원 동기 소재 (회사별)

- IT 대기업:
- 스타트업:
- 금융/보험:
```

## 에러 처리

- 공고 URL 접근 불가 --> "URL 접근 실패. 질문 항목을 직접 입력해주세요."
- 프로필 문서 없음 --> 프로필 생성 대화 시작 또는 기존 자소서 폴백
- 글자 수 초과 --> 자동 축약 후 축약 내용 안내
- 피드백 루프 5회 초과 --> 현재 상태로 확정 + 미달 항목 WARNING
- 문체 참고 글 없음 --> 일반 20대 취준생 톤 + [TODO] 마커 증가

## 참고 사항

- AI 탐지 회피 상세 가이드: `references/anti-ai-detection.md`
- 프로필 문서는 자소서를 쓸수록 풍부해짐 (새 경험 추가 권장)
- 같은 항목을 여러 회사에 제출 시, 기존 자소서를 참고하여 회사별 맞춤 변형 가능
