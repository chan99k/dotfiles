# AI Detective: Fix (감점 분석 + 개선안)

직전 `/ai-detective:check` 또는 `/ai-detective:deep` 결과를 기반으로, 목표 점수까지 도달하기 위한 구체적 개선안을 제시하는 커맨드.

## 인수

| 형식 | 설명 | 예시 |
|------|------|------|
| 목표 점수 | 도달하려는 점수 (기본 95) | `/ai-detective:fix 95` |
| 없음 | 기본 목표 95점으로 실행 | `/ai-detective:fix` |

## 전제 조건

- 같은 세션에서 `/ai-detective:check` 또는 `/ai-detective:deep`이 먼저 실행되어 있어야 함
- 실행되지 않았으면 사용자에게 먼저 체크를 실행할지 확인

## 실행 절차

### 1. 직전 평가 결과 수집

같은 대화에서 수행된 AI 탐지 결과에서 다음을 추출:
- 각 전문가의 점수와 감점 요인
- AI 의심 패턴 목록 (LINE 번호 포함)
- 현재 평균 점수

### 2. 갭 분석

```
현재 평균: XX점
목표 점수: {인수 또는 95}점
필요 상승: +XX점
```

### 3. 감점 요인 분류 및 우선순위

감점 요인을 **영향도 순**으로 분류:

| 우선순위 | 유형 | 예상 점수 상승 | 난이도 |
|---------|------|---------------|--------|
| P1 | 구조적 문제 (표 과잉, 깔끔한 정리) | +5~8점 | 중 |
| P2 | 어휘/표현 (AI 관용구, 번역투) | +3~5점 | 하 |
| P3 | 톤/감정 (양비론, 분석적 톤) | +2~3점 | 중 |
| P4 | 참조/포맷 (체계적 분류) | +2~3점 | 하 |

### 4. 구체적 개선안 제시

각 감점 요인에 대해 **라인 번호와 함께** 구체적 수정 방향을 제시.

출력 형식:

```markdown
## 개선안 (현재 XX점 → 목표 XX점)

### P1: {감점 유형} (예상 +X점)

**문제**: {구체적 설명}

**[LINE XX-XX] 현재**:
> (현재 텍스트)

**수정 방향**:
> (개선된 텍스트 또는 방향 설명)

**이유**: {왜 이 수정이 점수를 올리는지}

---

### P2: ...

---

## 예상 결과
| 항목 | 현재 | 수정 후 예상 |
|------|------|-------------|
| 어휘/표현 | XX | XX |
| 구조/리듬 | XX | XX |
| ... | | |
| **평균** | **XX** | **XX** |
```

### 5. 수정 적용 확인

개선안을 제시한 후:

1. 사용자에게 **각 항목별로 적용 여부** 확인
2. 사용자가 승인한 항목만 파일에 반영
3. **절대로 사용자 확인 없이 파일을 수정하지 않음**

### 6. 재평가 안내

수정 적용 후:
- "수정 반영했습니다. `/ai-detective:check` 또는 `/ai-detective:deep`으로 재평가할까요?"
- 재평가 → 목표 미달 → fix 재실행의 **피드백 루프** 가능

## 주의사항

- 개선안은 **글의 핵심 내용을 바꾸지 않는 범위** 내에서만 제안
- 사용자의 개인적 경험, 고유한 표현, 의도적 스타일은 존중
- "AI처럼 보인다"는 이유만으로 좋은 표현을 삭제하지 않음
- 구조적 변경(표 → 산문 등)은 **대안을 함께 제시**하고 사용자가 선택하게 함
