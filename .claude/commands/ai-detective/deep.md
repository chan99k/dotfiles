# AI Detective: Deep Check (5명 전문가)

대상 파일의 AI 생성 여부를 5명의 전문가가 병렬로 정밀 평가하는 커맨드.

## 인수

| 형식 | 설명 | 예시 |
|------|------|------|
| 파일 경로 | 평가 대상 텍스트 파일 | `/ai-detective:deep /path/to/file.md` |
| 없음 | 현재 대화에서 언급된 파일 자동 감지 | `/ai-detective:deep` |

## 실행 절차

### 1. 대상 파일 확인

- 인수로 파일 경로가 주어지면 해당 파일을 Read
- 인수가 없으면 현재 대화에서 가장 최근에 작업한 텍스트 파일을 자동 감지
- 파일을 찾을 수 없으면 사용자에게 경로 요청

### 2. AI 탐지 가이드 로드

`/Users/chan99/.claude/skills/cover-letter/references/anti-ai-detection.md`를 Read하여 AI 특유 패턴 기준을 숙지.

### 3. 5명 전문가 병렬 평가

Agent 도구로 5명의 전문가를 **동시에** 실행. 모델은 **sonnet** 사용.

#### 전문가 구성

| # | 전문가 | 관점 | 평가 기준 |
|---|--------|------|----------|
| 1 | **언어학 박사** | 어휘/표현 패턴 | AI 관용구("살펴보겠습니다", "다양한"), 번역투("~을 통해"), 구어체 자연스러움, 어휘 다양성, 개인적 경험 표현 |
| 2 | **컴퓨터 언어학자** | 문장 구조/리듬 | 문장 길이 분포(짧은/중간/긴 불규칙성), 문단 구조 패턴, 접속사 배치 규칙성, 정보 밀도 균일성, 톤 전환 자연스러움 |
| 3 | **테크니컬 라이터** | 논리 전개/서사 | 경험 기반 서사 vs 일반론, 시행착오 과정, 비선형적 사고("아 근데..."), 의견/팩트 구분, 예상치 못한 전개, 불완전함의 자연스러움 |
| 4 | **도메인 전문가** | 기술적 진정성 | 구체적 디테일(프로젝트명, 에러 메시지), 기술적 정확성, "이론 vs 실제 사용" 차이, 도구 사용 디테일, 의사결정 과정, 불완전한 해결 |
| 5 | **디지털 포렌식 전문가** | 감정/개성 | 감정 변화 흐름(호기심→놀람→실망→결심), 유머/위트 자연스러움, 좌절/짜증, 자기 비하, 독자와의 대화, 억지 긍정 여부, 필자 말버릇 |

**도메인 전문가 역할 맞춤**: 대상 글의 도메인에 맞게 전문가 4의 역할을 조정.

- 기술 블로그 → "시니어 개발자 (10년 경력)"
- 자기소개서 → "대기업 인사담당자 (채용 경험 15년)"
- 이력서/포트폴리오 → "IT 헤드헌터 (기술 직군 전문)"

#### 각 전문가에게 전달할 프롬프트 구조

```
당신은 AI 생성 텍스트 탐지 전문가({역할})입니다.
아래 파일을 읽고, **{관점}** 관점에서 AI 생성 여부를 평가해주세요.

파일: {파일 경로}

## 평가 기준
{해당 전문가의 구체적 기준}

## AI 탐지 가이드 참조
{anti-ai-detection.md의 관련 섹션}

## 출력 형식
### {관점} 분석

#### AI 의심 패턴
- [LINE] "표현" — 이유

#### 사람다운 패턴
- [LINE] "표현" — 이유

#### 점수: XX/100
(100 = 완전히 사람이 쓴 글, 0 = 완전히 AI가 쓴 글)

#### 근거 요약 (2-3줄)
```

```markdown
| 전문가 3 | 논리/서사 | XX/100 | 사람/AI의심 |
| 전문가 4 | 기술적 진정성 | XX/100 | 사람/AI의심 |
| 전문가 5 | 감정/개성 | XX/100 | 사람/AI의심 |
| **평균** | | **XX/100** | |

### 전문가 공통 "사람다운" 증거
- (2명 이상이 언급한 사람다운 패턴)

### 전문가 공통 "AI 의심" 요소
- (2명 이상이 언급한 AI 패턴)

### 의견이 갈린 부분
- (전문가 간 판단이 다른 부분 + 각자 근거)
```

### 5. 후속 안내

- 평균 점수와 목표(기본 95점) 간 갭이 있으면: `/ai-detective:fix {목표점수}`로 개선안 제시 가능함을 안내
- 특정 전문가 점수가 낮으면 해당 관점의 구체적 개선 방향 1줄 요약
