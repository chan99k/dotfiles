# AI Detective: Quick Check (3명 전문가)

대상 파일의 AI 생성 여부를 3명의 전문가가 병렬로 빠르게 평가하는 커맨드.

## 인수

| 형식 | 설명 | 예시 |
|------|------|------|
| 파일 경로 | 평가 대상 텍스트 파일 | `/ai-detective:check /path/to/file.md` |
| 없음 | 현재 대화에서 언급된 파일 자동 감지 | `/ai-detective:check` |

## 실행 절차

### 1. 대상 파일 확인

- 인수로 파일 경로가 주어지면 해당 파일을 Read
- 인수가 없으면 현재 대화에서 가장 최근에 작업한 텍스트 파일을 자동 감지
- 파일을 찾을 수 없으면 사용자에게 경로 요청

### 2. AI 탐지 가이드 로드

`/Users/chan99/.claude/skills/cover-letter/references/anti-ai-detection.md`를 Read하여 AI 특유 패턴 기준을 숙지.

### 3. 3명 전문가 병렬 평가

Agent 도구로 3명의 전문가를 **동시에** 실행. 모델은 **haiku** 사용.

#### 전문가 구성

| 전문가 | 관점 | 평가 기준 |
|--------|------|----------|
| **어휘/표현 전문가** | 어휘 패턴 | AI 관용구 남발, 번역투, 구어체 자연스러움, 어휘 다양성 |
| **구조/리듬 전문가** | 문장 구조 | 문장 길이 변화, 문단 구조 패턴, 접속사 배치, 정보 밀도 균일성 |
| **톤/개성 전문가** | 감정/개성 | 감정 변화 자연스러움, 자기 비하/유머, 필자 개성, 억지 긍정 여부 |

#### 각 전문가에게 전달할 프롬프트 구조

```
당신은 AI 생성 텍스트 탐지 전문가입니다.
아래 파일을 읽고, {관점} 관점에서 AI 생성 여부를 평가해주세요.

파일: {파일 경로}

## 평가 기준
{해당 전문가의 구체적 기준 - 위 표 참조}

## AI 탐지 가이드 참조
{anti-ai-detection.md 내용 중 관련 부분}

## 출력 형식
### {관점} 분석

#### AI 의심 표현 (있으면 나열)
- [LINE] "표현" — 이유

#### 사람다운 표현 (있으면 나열)
- [LINE] "표현" — 이유

#### 점수: XX/100
(100 = 완전히 사람이 쓴 글, 0 = 완전히 AI가 쓴 글)

#### 근거 요약 (2-3줄)
```

### 4. 종합 리포트 출력

3명의 결과를 취합하여 아래 형식으로 출력:

```markdown
## AI 탐지 Quick Check 결과

| 전문가 | 관점 | 점수 | 판정 |
|--------|------|------|------|
| 전문가 1 | 어휘/표현 | XX/100 | 사람/AI의심 |
| 전문가 2 | 구조/리듬 | XX/100 | 사람/AI의심 |
| 전문가 3 | 톤/개성 | XX/100 | 사람/AI의심 |
| **평균** | | **XX/100** | |

### 주요 감점 요인
- (감점 요인 요약)

### 사람다운 증거
- (사람다운 패턴 요약)
```

### 5. 후속 안내

- 평균 90점 미만이면: `/ai-detective:fix {목표점수}`로 개선안 제시 가능함을 안내
- 더 정밀한 검사가 필요하면: `/ai-detective:deep`을 안내
