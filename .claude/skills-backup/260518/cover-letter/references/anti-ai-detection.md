# AI 작성 탐지 회피 가이드

## AI가 쓴 글의 특징 (제거해야 할 패턴)

### 1. 구조적 패턴
- 모든 단락이 동일한 길이
- "첫째~둘째~셋째" 또는 "먼저~다음으로~마지막으로" 반복
- 각 단락이 주제문+부연+결론의 동일 구조
- 과도하게 깔끔한 논리 전개

### 2. 어휘적 패턴
- "다양한", "폭넓은", "심도 있는", "유기적" 등 AI 선호 형용사 남용
- "~를 통해", "~를 바탕으로", "~에 기반하여" 과도한 사용
- "성장", "도전", "열정", "소통" 등 추상적 키워드 반복
- "이를 통해 ~을 배웠습니다" 식의 정형화된 결론
- "~한 경험이 있습니다", "~에 기여했습니다" 식 수동적 종결

### 3. 톤/감정 패턴
- 감정의 굴곡 없이 일정한 톤
- 모든 경험을 긍정적으로만 서술
- 실패나 좌절을 "성장의 발판"으로만 해석
- 지나치게 겸손하거나 지나치게 자신감 있는 균일한 톤

### 4. 내용 패턴
- 구체적 숫자/날짜/이름 없이 추상적 서술
- "~프로젝트에서 ~역할을 맡아 ~를 달성했습니다" 공식
- 회사에 대한 정보가 홈페이지 그대로 인용

## 자연스러운 글로 바꾸는 전략

### 1. 불균형한 구조
- 단락 길이를 의도적으로 다르게 (2줄짜리 + 5줄짜리 + 3줄짜리)
- 간혹 짧은 문장 하나로 단락 구성 ("그때 처음 알았습니다.")
- 결론이 깔끔하지 않아도 OK (실제 사람은 깔끔하게 안 마무리함)

### 2. 구체성 주입
- 날짜, 시간, 장소, 사람 이름, 구체적 숫자
- 감각적 디테일 ("새벽 3시 카페에서", "빨간 에러 메시지를 보며")
- 개인적 반응 ("솔직히 당황했습니다", "자존심이 상했습니다")

### 3. 구어체 자연스러움
- 완벽한 문어체가 아닌 약간의 구어체 혼합
- "~거든요", "~했는데요" 같은 종결어미 가끔 사용 (과하지 않게)
- 괄호 안 부연 설명 활용 ("(지금 생각하면 무모했지만)")
- 접속사 줄이기 ("그리고", "또한", "더불어" 대신 문장 끊기)

### 4. 감정의 진폭
- 실패 경험을 솔직하게 (즉시 교훈으로 전환하지 않기)
- 불확실함 표현 ("아직도 정답인지는 모르겠지만")
- 개인적 동기 노출 ("솔직히 처음엔 취업 스펙을 위해 시작했습니다")

### 5. 문체 다양화
- 같은 종결어미 3번 연속 금지 ("~습니다. ~습니다. ~습니다." X)
- 문장 길이 변화 (10자짜리 짧은 문장 + 50자짜리 긴 문장)
- 의문형, 감탄형, 인용 등 다양한 문장 유형 섞기

## 문체 학습 방법

사용자의 기존 글(자소서, 블로그, 에세이)이 있을 경우:
1. 글에서 자주 쓰는 종결어미 패턴 추출
2. 문장 평균 길이와 분포 파악
3. 선호하는 접속사/연결어 파악
4. 감정 표현 방식 (직접적 vs 간접적) 파악
5. 이 패턴을 자소서에 반영

기존 글이 없을 경우:
- 20대 한국 남성 IT 취준생의 일반적 문체 적용
- 수정 마커 `[TODO: 본인 경험으로 대체]`를 남겨 직접 수정 유도
